# Expected Answers and Baselines

This document outlines the standard of quality for answers generated by the RAG system. AI agents generating test cases or evaluating system performance must use these baselines to measure correctness, tone, and citation density.

## Core Requirements for a "Perfect" Answer
1. **Directness:** Answers the user's question in the first sentence.
2. **Attribution:** Every technical claim must be immediately followed by an inline citation e.g., `[Spanner, 2012, p. 4]`.
3. **Completeness without Fluff:** Covers all necessary nuances but avoids redundant introductory remarks (e.g., "That's a great question about databases!").
4. **Formatting:** Uses bolding for key terms and markdown tables for comparisons.

## Example Baseline 1: Comparative Query
**Query:** "Compare the consistency models of Dynamo and Spanner."
**Expected Answer Structure:**
- **Dynamo (Eventual Consistency):** Explains vector clocks, sloppy quorums, and anti-entropy (Merkle trees) used to achieve high availability at the cost of strong consistency `[Dynamo, 2007, p. 3-4]`.
- **Spanner (External Consistency/Linearizability):** Explains TrueTime, atomic clocks, and commit wait times used to provide globally synchronous transactions `[Spanner, 2012, p. 7]`.
- **Comparison Table:** A markdown table summarizing availability, consistency guarantees, and clock dependencies.

## Example Baseline 2: Definition Query
**Query:** "What is the CALM theorem?"
**Expected Answer Structure:**
- **Definition:** CALM (Consistency As Logical Monotonicity) states that a program has a consistent, coordination-free distributed implementation if and only if it is monotonic `[CALM Theorem, 2012, p. 2]`.
- **Explanation:** Clarifies that monotonic programs only add to their state (no retractions), meaning they do not require distributed locks or consensus protocols to ensure consistency `[CALM Theorem, 2012, p. 4]`.

## Handling "I don't know" Scenarios
If the retrieval engine returns chunks that do *not* answer the question (e.g., querying for a proprietary system not in the database like Snowflake), the expected answer is:
"I cannot answer this question based on the provided research papers. The context does not contain information about [Topic]."
**Never** fallback to the LLM's pre-trained weights.
