# System Design

This document outlines the high-level system design of the Database Research RAG platform. AI agents modifying or extending the system must adhere to these structural principles.

## Core Tenets
1. **Source of Truth:** The raw PDFs (`/data/raw_pdfs/`) are the immutable source of truth. All derived data (chunks, embeddings, parsed JSON) must be reproducible from the PDFs.
2. **Modularity:** The system is divided into isolated services (Ingestion, Retrieval, LLM). Each service should be independently testable and deployable.
3. **Traceability:** Every answer generated by the LLM MUST be traceable back to a specific PDF, page number, and text chunk.

## Components

### 1. Data Ingestion Pipeline (`services/ingestion`)
- **PDF Parser:** Extracts text, tables, and metadata (Title, Authors, Year) from PDFs. Handles multi-column layouts common in academic papers.
- **Chunker:** Splits text into semantically meaningful units (e.g., paragraphs, sections). Preserves document hierarchy.
- **Embedder:** Converts text chunks into dense vector representations using a pre-trained sentence transformer model (e.g., `all-MiniLM-L6-v2` or OpenAI's `text-embedding-3-small`).

### 2. Knowledge Store
- **Vector Database (OpenSearch/Pinecone):** Stores vector embeddings for semantic search.
- **Document Store (S3/DynamoDB):** Stores the full parsed text chunks, metadata, and references for exact keyword search (BM25) and retrieval during generation.

### 3. Retrieval Engine (`services/retrieval`)
- **Query Analyzer:** Processes the user's natural language query, expanding acronyms (e.g., LSM to Log-Structured Merge) or extracting keywords.
- **Hybrid Searcher:** Executes parallel searches: Vector Search (Semantic) and Keyword Search (BM25). Combines results using Reciprocal Rank Fusion (RRF).
- **Reranker:** Uses a Cross-Encoder to score the top-K combined results against the query, filtering out irrelevant chunks.

### 4. LLM Orchestration (`services/llm`)
- **Prompt Builder:** Constructs the prompt by injecting the user query and the top-ranked retrieved chunks. Enforces formatting rules (citations).
- **Response Generator:** Calls the LLM API (e.g., OpenAI GPT-4, Anthropic Claude 3) to generate the answer based *only* on the provided context.

## Deployment Architecture
- **AWS Infrastructure:**
  - **S3:** Raw PDFs and parsed JSON objects.
  - **Lambda/ECS:** Ingestion tasks and API endpoints.
  - **OpenSearch Serverless:** Managed vector and text search.
  - **API Gateway:** Entry point for UI and external clients.
